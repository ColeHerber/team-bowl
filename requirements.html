Requirements

What the robot must do

Every requirement has a priority flag: H necessary for function, M greatly improves function, L minor improvement. Validation status is from the System Validation Review.

R-1 — Safety

R-1.1
Virtual Emergency Stop (E-Stop) H
Robot must include a wireless emergency stop that pauses all function as soon as possible.
E-Stop command must bring the robot to a full stop within 1 second.
✓ Pass
R-1.2
Physical Emergency Stop H
Robot must include a physical emergency stop switch that pauses all function as soon as possible.
Activation must bring the robot to a full stop within 1 second.
✓ Pass
R-1.3
External Status Indicators M
Robot must display current operational mode/status via lights or audio cues.
Status indicators must be visible/audible from 15 meters away (per OSHA/IEC 60204-1).
✓ Pass

R-2 — Physical Dimensions

R-2.1
Weight Limits H
Robot must be a size moveable by the average person. Users will need to transport the robot between uses.
Robot plus cargo must be less than 60 lbs (OSHA lifting recommendation).
✓ 57.94 lbs
R-2.2
Dimensions M
Robot must be a size usable for transport and storage. Users will need to store it between uses.
Robot must be less than 24 × 24 × 24 inches.
✓ Pass

R-3 — Carrying Capacity

R-3.1
Cargo Storage Volume H
Robot must be able to transport an acceptable size of objects.
Robot must have a 5-gallon minimum cargo capacity.
✓ 20.6 qt
R-3.2
Minimum Cargo Weight Capacity M
Robot must be able to transport an acceptable weight of objects.
Robot must be able to store a minimum of 15 lbs.
— Inspection

R-4 — Operational

R-4.1
Indoor/Outdoor Operation H
Robot must operate as expected indoors and on some rougher sidewalks.
Robot successfully navigates indoor and outdoor test courses.
✓ Pass
R-4.2
Simulation Environment L
Robot must have a simulation twin with mock environments.
Simulation model must replicate and perform well in response to common challenges.
✓ Pass

R-5 — Terrain Traversal Ability

R-5.1
Terrain Traversal H
Robot must be able to traverse difficult terrain to suit user needs.
Robot must successfully follow the user over assumed difficult terrain in a trial run.
✓ Pass
R-5.2
Obstacle Avoidance H
Robot must not get stuck on obstacles while following the user.
Robot does not become unable to move for more than 5 seconds while following users in trial runs.
✓ Pass
R-5.3
Move Multiple Directions from Rest M
Robot must be able to move multiple directions from rest to avoid immediate obstacles.
Robot must have a minimum of two full degrees of freedom at rest.
✓ Pass
R-5.4
Locking Cargo Enclosure L
Robot must have a locking cargo enclosure to prevent spillage on rough terrain.
Lid must physically lock with a minimum gap of 0.25 inches on all sides.
✓ Pass

R-6 — User Operation

R-6.1
Autonomous Operation H
Robot must be fully functional without human interaction. Requiring constant inputs makes this less desirable.
Robot must be operational with 0 humans providing it with direct inputs.
~ Partial
R-6.2
Tele-Operation H
Robot must be able to be controlled with direct inputs from the user.
Every function the robot can carry out autonomously can also be executed by direct user command.
✓ Pass
R-6.3
Robot Speed H
Robot speed must be less than the legal maximum for the City of Pittsburgh to operate on public sidewalks.
Robot must be under 12 MPH (City of Pittsburgh ordinance).
✓ Pass

R-7 — Designated User Following

R-7.1
Autonomous User Tracking H
Robot must automatically track user via an onboard tracking system.
Robot must be able to locate user position within ±0.5 meters.
✓ Pass
R-7.2
Following Distance Range M
User tracking must work within a reasonable range.
Robot must track user between a minimum of 0.5 meters and a maximum of 5 meters.
✓ Pass

R-8 — Robot Range

R-8.1
Operation Duration M
Robot must be fully operational for a usable amount of time.
Robot must have an operation duration always greater than 30 minutes.
✓ Pass
R-8.2
Operation Range M
Robot must be fully operational for a reasonable travel distance.
Robot must have a 2-mile operation range.
— Not tested

R-9 — Reliability & Maintainability

R-9.1
Drop Resistance H
Robot must survive reasonable drops without sustaining critical damage.
Robot can function as designed after a 0.5-meter drop.
✓ Pass
R-9.2
Maintainability H
System must allow for easily replaceable modular components to ensure fast maintenance.
All components must be replaceable within 15 minutes.
✓ Pass
R-9.3
Assault Resistance M
Robot must continue expected function after side impacts from human actor bumps in crowded environments.
No loss in functionality when subject to a 150-pound lateral impact.
✓ Pass
R-9.4
Floor Protection M
Robot must not leave marks on indoor flooring to ensure suitability for indoor use-cases.
Robot does not leave marks on the floor after a 30-minute trial run.
✓ Pass

R-10 — Economic

R-10.1
Price Cap H
Build cost must remain under $2,500 to ensure development feasibility given budget constraints.
Summed BOM cost must be less than or equal to $2,500.
✓ $2,446
R-10.2
Manufacturability M
Must be manufacturable with 3-axis CNC, lathing, and 3D printing using available resources.
BOM shows suitable fabrication method for all parts.
✓ Pass

R-11 — Connectivity

R-11.1
Wireless Communication H
Robot must be able to communicate wirelessly so users can operate it without a wired connection.
Robot must be communicable over an FCC-allowed wireless method.
✓ WiFi

R-12 — Social & Human Factors

R-12.1
Aesthetic Consistency H
Robot must use a consistent color scheme to increase acceptance and trust in public settings.
Exterior structure must show uniform color scheme.
✓ Black & red
R-12.2
User Accessibility M
Robot must be equipped with an ergonomic cargo lid handle to accommodate a variety of human actors.
Average time to retrieve or deposit cargo is less than 15 seconds across a diverse user test group.
✓ <15 sec
